      Release 4.0.0 February 20 2023

      Major new features with this release:

      • The ModelStore pattern has been clarified and articulated as a collection of open format models with associated catalog and finding aids. These changes support new workflows for model validation and in future releases will support the way that the Model Downloads site is updated.
      • The ModelBatch pattern has been articulated as an extension of the more general Workspace pattern. These improvements provide clarity to the workflows for Major Updates, for model validation and for model splitting and repairs.

        Several workflows, tools and tasks associated with the ModelBatch pattern have been improved , including:

        • An autoAssembled_MP stage for the consolidated source models as multipatch. This is the new target for Assigning Model and Geometric attributes.
        • Batch-specific tool-box provides easily repeatable source model assembly and assignment of batch-specific model attributes.
        • The Assign Geometric Attributes tool and task has been simplified. It now modifies the input feature class.
        • Batch-specific ModelStore preserves OBJ models
        • Two new FME workbenches:
          • Validate assembled models produce a ModelStore representing validated models representing source models and changes performed in the initial Prep_MP.
          • Produce a Ready_MP feature class validated as proving a round-trip to open OBJ format.
        • The Assign Model Attributes tool automatically detects the Model Batch and sets various dates and editor information.
        • Promo/Demo for new and modified models is now carried out in the Prep_MP feature class

        New tools in pbcUtils tool group

        • Import Models with model-specific CSV Insertion points. This is a common way of exchanging 0,0,0 based 3D models between platforms.
        • Import Models frm KMZ format. Another common exchange format -- which is similar to our ModelStore idea

    Changes to Schema and related Workflows

    Schema changes expand the list of QA Flags to better support Promo/Demo workflows.

    • The Modified and Demolished status have been dropped and merged into the status "History". For any model that represents a past shape for a structure, its status simply becomes History. If the Disappear Date is left Null, this date wil be set automatically when the next geodatabase edition is pushed.
    • Models with status: Under Construction and Complete will now remain in the Approved feature class until they are superseded by measured models. This wil lbe more compatible with the lifecycle of hand-made models which stay in the Change Model Store until they are superseded by measured models.
    • The status-driven symbolization and group layers and definition query views have been updated to reflect the consolidation of the history layers.
    • The status filtering procedures for pushing new editions of web scenes and geodatabases are being updated to reflect the new behavior of Approved, Under Construction and Complete.
    • Several new values have been added to the QA_Flag domain to support workflows for flagging Promo and Demo candidates and various cases of models needing repair.
    • The Model ID has been expanded to a 21 character string in the form: BOS-A1B2C-3D4E5-F6G7H. This new ID has a 1 in 2 billion chance of drawing a duplicate in a model collection of 50 million models. See Model Identifiers.
    • Project_ID will change from an Integer type to a Character string with 20 characters.
    • A new ModelStore field has been added which wil make it possible to construct a path or URL to the OBJ for any model, some of which may be in the Change, Internal, or As-Built ModelStores. A domain has been added to regulate the values of the ModelStore field. Domain management tools updated to set this up on MP FCs.
    • Hand-Made models of buildings that have been observed to be "topped out" will now have their status set to Construction Complete. Models with this status will stay in the Approved_MP feature class until they are replaced by a measured model. Whence their status will become Project Complete which will cause them to be moved to the Internal feature class and model store. This change requires an update to the Sort Next GDB Edition tools.
    • Created new domain for StructType.

    May 20, 2022: CityTiler Repository Posted

    The CitySchema-CityTiler project includes all of the tools for:
    • Creating a tiling scheme that facilitates exchange of georeferenced models between GIS and design-oriented 3d modeling tools.
    • Creating tiled terrain models that integrate very detailed triangulated models witin the boundaries of the city's photogrammetry. Filling in areas outside the city limit with details from the MassGIS state-wide DTM.
    • Creating tiled groundplan images that drape precisely onto the tiled terrain models
